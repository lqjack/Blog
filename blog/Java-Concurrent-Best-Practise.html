可变状态是至关重要的

  • 可变状态越少，越容易保证线程安全
  • 尽量将域声明为final
  • 不可变对象一定是线程安全的
  • 封装有助于管理复杂性
  • 锁控制可变变量
  • 保证同一个不变性条件的所有变量时，需要使用锁
  • 进行复合操作需要锁
  • 禁止随意推断不需要加锁
  • 同步策略文档化

同步工具类

  1. 栅栏Barrier CycleBarrier Exchange

闭锁用于等待事件，而栅栏用于等待线程。

  1. 闭锁CountDownLatch FutureTask

	while(当前状态不允许操作) {
		if(需要阻塞获取请求) {
			如果当前线程不在队列中，则将其插入队列
			阻塞当前线程
		}
		eles
			返回失败
	}
	可能更新同步器状态
	如果线程位于队列中，将其移除队列
	返回成功
}

void release() {
	更新同步器状态
	if(新的状态允许某个被阻塞的线程获取成功) {
		解除队列中一个或者多个阻塞状态
	}
}
